## D7 — Startup partial candle: discard the first bucket (owner product decision)

**Decision.** The Candle Builder discards each symbol's **first bucket after process start** at rollover — it is never published and never persisted. The first published candle is the first **fully observed** 1-minute candle. The missing minute is not fabricated and not backfilled.

**Why.** Architecture §4.1 defines only steady-state bucketing; the gate's reconciler flagged the startup minute (observation begins mid-minute → wrong o/l/v) being persisted as truth. Owner ruling: **correctness over completeness** — a partially observed candle is false data.

**Consumers.** `core/candle_builder.py` (P0.12); builder tests prime each symbol with a throwaway prior-minute trade.

## D8 — CORS policy for the standalone frontend

**Decision.** `create_app()` adds FastAPI's `CORSMiddleware`: `allow_origins=["*"]`, methods `GET`+`POST`, headers `Authorization` only, credentials **off**. The D3 bearer token remains the only access gate.

**Why.** The frontend is standalone static files (§9; P0.22 owner refinement) and is therefore always a foreign origin to the API. The documented `file://` mode sends the literal unpinnable origin `null` (any sandboxed iframe also sends `null`, so allowlisting it equals `*` — OWASP guidance), and a static-host origin is deployment-specific. Evidence: captured preflight `OPTIONS /candles` with `Origin: http://127.0.0.1:8080`, `Access-Control-Request-Headers: authorization` answered 405 before the fix. With no cookies/credentials in the auth design, the origin value protects nothing — the token does.

## D5 clarification — n_trades carries raw-trade counts

**Decision.** Normalized `Trade` gains `n_trades: int = 1` = the number of **raw trades** the event aggregates (Binance aggTrade trade-ID span `l − f + 1`); the builder **sums** it. Candle `n_trades` therefore matches the official kline `"n"` semantics, and tick-built rows agree with kline-sourced bootstrap/backfill rows — one column, one meaning. The D5 comparison rule itself is unchanged.

**Why.** The gate exposed a systematic mismatch: the builder counted aggTrade *events* (e.g. 637) against the kline's raw count (2,927). D5's 0.1% tolerance assumed both sides counted the same thing; they did not.

## Windows development compatibility note

`asyncio.add_signal_handler` is Unix-only; `main.py` guards it with `try/except NotImplementedError` so the composition root also runs on the Windows dev machine (uvicorn falls back to `signal.signal`, Ctrl+C only). The production Linux/systemd path (D4) is byte-for-byte unchanged. The SIGTERM graceful-shutdown test remains POSIX-only (`skipif`, documented in `tests/test_main.py`) because Windows `send_signal(SIGTERM)` hard-terminates instead of exercising the handler.
