# DECISION P1.12 — Trendline Engine detail set (A4 + A5 + Step 1–5 constants)

**Date:** 2026-07-18 · **Status:** Accepted (owner approval via the P1.13 go-ahead referencing "the accepted D11") · **Scope:** implementation decision only — zero architecture change.
This document is the **sole specification** for P1.13–P1.18: every constant and behavioural rule §4.3 left open is frozen here. Implementation makes zero further choices.

---

## Objective

§4.3 gives the five-step trendline algorithm with several unpinned details: the `levels` row semantics for lines (A4), the log-space/price-space tolerance reconciliation (A5), the scoring constants, archive semantics, the timeframe, and the break/fake-break episode mechanics. All are frozen below.

## Motivation

Trendlines are S3's signal source (P3.15), a confluence input (P2.18), and the subject of the P1.22 audit gate. Ad-hoc choices during coding would be uncalibratable and unauditable; freezing them keeps P1.13–P1.18 a pure transcription, per the project's decision discipline.

## Final accepted decisions (D11)

### D11.1 — Row semantics (A4, restating what migration 002 already froze)
`levels` row for a line: `kind='TRENDLINE'`, `p1` = price at `t1`, `p2` = price at `t2` (anchor pivot prices, linear), `slope` stored **redundantly** as the log-space per-bar slope (D11.4), `created_ts` = acceptance bar. Lifecycle columns (`touches`, `status`, `status_ts`) transition via the existing `update_level_lifecycle` helper only. Persistence remains **capability-only in Phase 1** per the approved R1 ruling — no production wiring, tx-isolated tests only.

### D11.2 — Timeframe rules
Candidates come from **1m confirmed pivots only** (the P1.5 detector's 1m stream); tolerance and body thresholds use **ATR(14, 1m)**; `levels.tf = '1m'`. 5m pivots serve A8 external structure exclusively. There is no 5m trendline engine in v1.

### D11.3 — Time axis
All line arithmetic runs on **positional bar indices** of the 1m stream (consistent with the pivot detector's positional stance; gaps neither stretch nor shrink a line). Timestamps (`t1`, `t2`) are identity and persistence fields only. The engine numbers bars internally from its own stream start.

### D11.4 — Geometry (A5): log-price space
For anchors `(a, b)`, `b` newer: `slope = (ln(p_b) − ln(p_a)) / (idx_b − idx_a)` per bar; line value at bar `t`: `y(t) = ln(p_a) + slope × (t − idx_a)`. Support lines are built from L-pivot prices, resistance from H-pivot prices.

### D11.5 — Tolerance rule (A5, Part B resolution verbatim)
`tol_log(t) = 0.15 × ATR(14,1m at bar t) / close(t)` — the price-space 0.15×ATR band converted to log space at the evaluation candle. One unit-tested helper; every tolerance comparison in Steps 2–4 uses it. While ATR is unwarm the engine is entirely inactive (refuse-until-ready, A3 convention).

### D11.6 — Candidate universe (Step 1)
Last **N = 12** confirmed same-kind 1m pivots per side (sliding; oldest drops). All ordered pairs → ≤66 candidates per side, **regenerated only when a new confirmed same-kind pivot arrives** (P1-SPEC pin). Validity filters, in order:
1. **Direction:** support requires `slope > 0` (strictly ascending), resistance `slope < 0` (strictly descending) — §4.3's "ascending support / descending resistance" verbatim. Slope-zero (equal anchors) is excluded: horizontal structure is the Liquidity Engine's EQH/EQL territory (P2.9), avoiding double representation.
2. **No close cuts the anchor segment:** at every bar strictly between the anchors, the close must not lie strictly beyond the line (support: `ln(close) < y(t)` invalidates; resistance mirrored).

### D11.7 — Touch counting (Step 2)
Evaluated from the older anchor's bar to the current bar, one touch max per candle: touch iff `|ln(extreme) − y(t)| ≤ tol_log(t)` (extreme = low for support, high for resistance) **and** the close does not cross the line (support crossed iff `ln(close) < y(t)`, strict; mirrored for resistance). **Anchors count as touches** (their extremes lie on the line, distance 0) — so "≥3 touches" requires at least one non-anchor validation. Kept lines: touches ≥ 3.

### D11.8 — Scoring constants (Step 3)
`score = touches × 2 + span_bars / 20 − age_penalty` where:
- `span_bars` = `idx_b − idx_a` (bar distance between the anchors);
- `age_penalty` = `bars_since_last_touch / 100` — an explicitly **arbitrary, uncalibrated** starting constant (like D9's 0.1), owned by P5 calibration.
**Cluster dedup:** near-parallel iff `|s₁ − s₂| < 0.10 × max(|s₁|, |s₂|)` **and** line values at the current bar differ by less than `0.3 × ATR / close` in log space (the D11.5 transform with 0.3). Greedy, deterministic: sort by score descending (ties: newer `b`, then newer `a`), take the best, discard everything near-parallel to it, repeat. Cap: **3 support + 3 resistance** survive.

### D11.9 — Break episodes (Step 4)
Evaluated per closed candle against each kept line:
- **TOUCH:** the D11.7 touch condition → touch count increments (lifecycle column).
- **Close-through:** any close strictly beyond the line opens a **3-candle re-entry watch** (status stays `active` during the watch; the line accrues no touches meanwhile).
- **FAKE BREAK:** a close back on the valid side within the 3 candles → FAKE BREAK classification; the line stays `active` (the trap validated it).
- **BREAK confirmed:** no re-entry within 3 candles → the line transitions to `status='broken'`. The episode is classified **BREAK** (breakout setup, S3/P2.18 relevance) only if the *breaking* candle had `body > 0.8 × ATR` **and** RVOL ≥ 1.5; otherwise the line still breaks but no BREAK event is emitted (a non-qualifying breakout is a lifecycle fact, not a setup).
- **RVOL placeholder (roadmap P1.16):** until P2.2 swaps in real RVOL, the volume condition evaluates **True** via a code-flagged placeholder — chosen because a False placeholder would make BREAK unreachable and FAKE BREAK untestable in P1. P2.2 re-runs determinism after the swap, per Part D.

### D11.10 — Lifecycle, role-flip, archive (Step 5)
- **Role-flip:** when a line breaks, the same geometry is re-registered as a **new** opposite-side line (new `levels` row; the broken row keeps `status='broken'`): exempt from the D11.6 direction filter (its legitimacy is the flip, §4.3 verbatim), touch count restarts at 0 and accrues from the break bar onward; it joins the kept set only upon reaching ≥3 touches, and competes in scoring/dedup like any line.
- **Archive:** a kept line untouched for **300 bars since its last touch** → `status='archived'` (terminal). `'archived'` is added to the app-layer status vocabulary — legitimate because migration 002 deliberately has no CHECK constraints and assigns validation to the application layer; this document records the addition.
- **Cap eviction:** a line pushed out of the kept 3+3 by scoring → `status='archived'` at the eviction bar (no stale actives in the table).
- **Channels:** a support+resistance pair with `|s₁ − s₂| < 0.08 × max(|s₁|, |s₂|)` and both ≥3 touches → channel object; **mid-line = log-space midpoint** of the two lines. Channels are **derived, state-only objects** (pairs of persisted TRENDLINE rows) — never persisted: the §3 `levels.kind` vocabulary has no CHANNEL value, and the data model is frozen.

## Edge cases (pinned)

- ATR unwarm → engine fully inactive (no candidates, no touches, no episodes).
- Equal-price anchors → slope 0 → excluded (D11.6); prices are always > 0 (crypto), so the log transform needs no guard.
- One candle can be a touch on several *different* lines, but at most one touch per line.
- A close-through during another line's watch is independent — watches are per line.
- The 12-pivot window sliding a candidate's anchor out does **not** retract an already-kept line (kept lines live by lifecycle, not by the candidate window).
- Restart → engine re-warms from scratch (memory-only), like every P1 component; persisted rows are history, not state.

## Examples

1. **Tolerance:** ATR = 60, close = 30 000 → `tol_log = 0.15×60/30000 = 0.0003` (≈ 0.03%; ≈ 9 price units at that level).
2. **Score:** touches = 4, span = 180 bars, last touch 40 bars ago → `4×2 + 180/20 − 40/100 = 8 + 9 − 0.4 = 16.6`.
3. **Dedup:** slopes 0.0010 and 0.00095 → Δ = 0.00005 < 0.10×0.0010 ✓ near-parallel; if their current-bar values differ by < 0.3×ATR/close in log space, the lower-scored one is discarded.
4. **Fake break:** support close-through at bar t; closes at t+1 below, t+2 back above the line → FAKE BREAK at t+2, line stays active. If t+1..t+3 all stay below → line `broken` at t+3; BREAK event only if bar t had body > 0.8×ATR (volume placeholder True).
5. **Role-flip:** that broken ascending support re-registers as an ascending resistance candidate (direction-exempt), touches from the break bar; kept once it earns 3.
