# DECISION P3.6 — Trade Qualification Engine detail set (D16, §6 pins)

**Date:** 2026-07-18 · **Status:** Accepted (engine-wise directive; pins per
the D9–D15 precedent) · **Scope:** implementation decision only — zero
architecture change. Sole specification for the Trade Qualification Engine
(§6; roadmap P3.2–P3.11), second half of the combined Confluence + Trade
Qualification delivery. Folds the placeholder-relevant parts of the two
roadmap [DECISION] tasks: P3.1 (gate wiring) and P3.6 (A13/A14 full rubric).

---

## D16.1 — Scope ruling

§6 defines this engine as **Stage 1 hard gates + Stage 2 weighted score**
over the current market state. Direction (Long/Short), entry zone, stop
loss, and invalidation are **S1–S3 strategy outputs** (§5; roadmap
P3.12–P3.16) and trade-planner outputs (§7; P3.17) — they are NOT produced
here (the owner's "only if defined by the roadmap" conditional resolves
them out). "Consume ONLY the Confluence output" is read per the owner's
read-only list: consume frozen-engine outputs (confluence included),
recompute nothing — §6's rubric explicitly requires sweep/CHOCH/trend/
momentum evidence, all consumed from the frozen chain.

Outputs per closed 1m candle: gate report, Data Integrity tier
(PASS/DEGRADED from G1–G2), component scores, weighted score, verdict
(`NO_SIGNAL` | `BELOW_THRESHOLD` | `TRADEABLE` | `A_PLUS`), rule-alignment
counts + display string, deterministic reasons trace. **ANY gate fail →
`NO_SIGNAL` and score/components = null in the result and payload** (§6:
score is never displayed on gate fail).

## D16.3 — Stage 2 rubric (P3.6/A13/A14 folded; all constants uncalibrated, P5-owned)

`score = 0.30×Structure + 0.30×Liquidity + 0.25×Volume + 0.15×Momentum`
(§0 rule 3 / §6 — frozen weights, never renormalized). Components 0–100.
Recency windows (bars, inclusive `age ≤ W`): `W_SWEEP = 20`,
`W_STRUCT = 20`, `W_MOM = 5` — explicitly uncalibrated pins (D9 doctrine).

**Liquidity (0.30)** — §6's own rubric, verbatim items:
- sweep of multi-touch pool **+40**: a `SweepEvent` with target EQH/EQL
  within `W_SWEEP` (pools are ≥2-touch by construction — the event does
  not carry size, and the pool is gone once swept, so any pool sweep
  qualifies; pinned reading).
- sweep with CHOCH confirm **+30**: a `SweepShift` within `W_SWEEP`
  (the frozen liquidity engine's own 3-candle window produced it).
- entry zone confluence ≥2 objects **+20**: a D15 zone with
  `count ≥ 2` whose interval is within 0.3×ATR (the frozen band constant —
  no new magic number) of the current close.
- opposing pool distance ≥ 1.5R **+10**: **not evaluable** — R needs an
  entry/SL plan (P3.17). Scores 0, flagged; component max 90 until then.

**Structure (0.30)** — A13 routing (trendline events → Structure rubric):
- established trend **+30**: `TrendState.state` is BULLISH or BEARISH.
- recent with-trend BOS **+30**: a `BosEvent` within `W_STRUCT` whose
  direction agrees with the current trend (UP↔BULLISH, DOWN↔BEARISH).
- no recent opposing CHOCH **+20**: no `ChochEvent` within `W_STRUCT`
  against the current trend (reversal warning absent). Scores 0 when the
  trend is RANGE/None (nothing to oppose ⇒ no alignment evidence).
- validated-trendline interaction **+20**: a `TrendlineEvent` of kind
  TOUCH or FAKE_BREAK within `W_STRUCT` (kept lines are ≥3-touch
  validated; BREAK is a structure-changing event, not alignment).

**Volume (0.25)** — **component = 0, flagged**: the Volume Engine
(P2.1–P2.7) is deferred. Conservative direction (score can only be
understated). Recorded consequence: with Volume absent and the Liquidity
R-item flagged, the maximum achievable score is
0.30×100 + 0.30×90 + 0.15×100 = **72.0**, so TRADEABLE (≥75) and A_PLUS
(≥85) are mathematically unreachable until the Volume Engine and the
planner land. No renormalization — the frozen weights stand; a verdict of
BELOW_THRESHOLD in this era is honest, not broken.

**Momentum (0.15)** — from the frozen momentum utilities (their first
composition consumers):
- regime **+40/+20/0**: expansion +40, normal +20, coil or None 0
  (§4.2: scalping favors expansion; compression = avoid).
- momentum shift **+30**: `MomentumState.momentum_shift` fired within
  `W_MOM`.
- body dominance **+30**: `body_dominance > 0.5` strict (conviction
  proxy; None → 0).

## D16.4 — Verdict, display (A14), reasons

- Verdict: any gate fail → `NO_SIGNAL`; else `score ≥ 85` → `A_PLUS`,
  `≥ 75` → `TRADEABLE` (inclusive, §6 "≥"), else `BELOW_THRESHOLD`.
- Two-tier display (A14 pin): `data_integrity` = `PASS` iff G1 AND G2
  pass else `DEGRADED`; `agreement` = `"{n} of {m} rules aligned"` where
  m = evaluable rubric items (currently 10: Structure 4 + Liquidity 3 +
  Momentum 3; the flagged R-item and the absent Volume items are not
  counted as evaluable) and n = items that scored > 0. Score in the
  payload is the raw float; formatting is the frontend's.
- Reasons = deterministic rule-trace strings, pinned constants: each
  aligned rubric item contributes `"✓ <item> (+P <component>)"`; on gate
  fail the reasons are the failing gates' `"✗ <gate>: <detail>"` lines
  only. No free text, no LLM (§8).

## D16.5 — Shape, composition, persistence

- `engines/qualification.py`: `GateResult` + `QualificationResult` frozen
  dataclasses and one `QualificationEngine` per symbol holding frozen
  references (trend, ATR, momentum, regime — the BosDetector reference
  precedent) plus its own recency/continuity bookkeeping (pure fold).
  `update(candle, ...)` receives the bar's transient events (BOS, CHOCH,
  trendline events, sweeps, shifts, confluence zones, spread, clock) and
  returns the result.
- Composition additions (main.py — not frozen): per symbol a 5m
  `IncrementalATR` (fed in `step_5m` before the 5m pivot chain),
  `MomentumState` (after the 1m ATR per the P1.2 pinned order),
  `RegimeClassifier` (after both ATRs); `book.refresh` events are now
  captured and fed to qualification; `_wire_structure_engines` subscribes
  `BookTicker` (latest per symbol → G2) and accepts an optional clock
  provider (live `main()` passes the sampler; replay/tests pass nothing).
  Payload gains `"qualification"`.
- **No persistence.** Signals rows are P3.18's (immutable row per emitted
  signal); this engine emits state, not signals.

## Freeze-audit record (2026-07-18)

**Two genuine defects found (independently by both the conformance and
adversarial auditors), fixed + regression-tested before freeze:**

1. The +20 "no opposing CHOCH" item was direction-blind — ANY recent
   CHOCH denied it, wrongly punishing the post-flip window where the
   flip's own CHOCH is aligned with the new trend. Fixed: a CHOCH opposes
   iff DOWN↔BULLISH / UP↔BEARISH against the *current* trend.
2. Last-event-only BOS/CHOCH memory could not evaluate D16.3's window
   quantifiers (existential for the with-trend BOS, universal for the
   opposing CHOCH): in double-flip windows a newer non-agreeing event
   shadowed an older in-window agreeing one. Fixed: full (bar, direction)
   deques pruned to `age ≤ W_STRUCT`.

**Behavior-identical readings, recorded:**

- **D9 config plumbing gap (owner-schedulable):** this delivery is the
  first composition of MomentumState/RegimeClassifier; they are
  instantiated with constructor defaults numerically identical to the D9
  pins, but the D9-promised config surface (`regime.*`/`momentum.*` keys)
  does not exist yet. Zero behavior impact today; landing the plumbing is
  an owed D9 obligation for its own task, not silently folded into this
  freeze.
- **Mid-step exception fragility (pre-existing, composition-level):**
  `_last_ts` is set before the engines run, so a frozen-engine raise
  mid-step would permanently skip that candle for all engines. No
  producible wrong output absent such a raise; unchanged behavior since
  the P2 guard landed.
- **G2 degenerate/crossed books:** `mid ≤ 0` → flagged pass carrying the
  replay detail string (unreachable for BTC/ETH majors); a crossed book
  (bid > ask) yields a negative spread that literally satisfies the
  frozen "spread < 0.05%".
- **Clock boundary:** exactly `|offset| == 2.0 s` passes via the frozen
  D6 `in_sync` (inclusive) while §6 reads "< 2 s" — D6's surface owns the
  boundary (measure-zero on a measured float).
- **Gate-fail agreement string** is `"gates failed — no score"` rather
  than the `{n} of {m}` format — deliberate: §6 forbids any score-derived
  display on gate fail.
