# DECISION P2.8 — Liquidity Engine detail set (A9 + §4.4 pins)

**Date:** 2026-07-18 · **Status:** Accepted (engine-wise directive: Part B's proposed resolution adopted verbatim; remaining details pinned per the D9/D11 precedent) · **Scope:** implementation decision only — zero architecture change.
Sole specification for the Liquidity Engine (roadmap P2.8–P2.13 traceability).

---

## D12.1 — Session map (A9, Part B resolution verbatim)

UTC sessions: **ASIA 00–08 · LONDON 08–13 · NY 13–21 · LATE 21–00** (the
previously unassigned hole). Recorded for P3: G3's default is *no strategy
allowed in LATE* unless config says otherwise (implemented at P3.3, not here).
Session identity is derived purely from candle `ts` — no wall clock.

## D12.2 — EQH/EQL pools (§4.4 clustering, pinned)

- Inputs: the last **20** confirmed labeled **1m** pivots per kind (window
  constant, explicitly arbitrary, P5-owned; the D11 12-window governs
  trendlines only).
- Recompute on each new confirmed same-kind pivot (candidates pattern);
  inactive while ATR(14,1m) is unwarm.
- **Greedy newest-first anchoring:** walk pivots newest→oldest; each
  unclaimed pivot anchors a cluster; membership = unclaimed same-kind pivots
  with `|p − anchor| < 0.1 × ATR(14,1m)` (strict, §4.4 literal, evaluated at
  recompute time).
- **≥ 2 members = a pool** (H pivots → EQH, L pivots → EQL). Pool price =
  arithmetic **mean** of member prices. `pool_strength = size / (1 +
  newest_member_age_bars / 1440)` — §4.4's "recency weight" pinned as a
  one-day (1440-bar) decay, explicitly uncalibrated, P5-owned; evaluated at
  recompute time.
- **Sweep latch identity = the pool's member-ts tuple:** a swept member set
  is terminally swept; a pool that gains a new member is a *new* identity
  (fresh liquidity) and is sweepable again.

## D12.3 — Key levels (§4.4, pinned)

Running H/L extremes are tracked per UTC **day** (00:00 boundary), **ISO
week** (Monday 00:00 UTC) and **session** (D12.1). At each boundary crossing
(driven only by candle `ts`): the completed day's extremes become PDH/PDL,
the completed week's PWH/PWL, and the completed session's extremes become
that session's H/L levels. **Sweep targets = active pools + PDH/PDL +
PWH/PWL + each session's last completed H/L.** Running (current) extremes
are exposed as state but are never sweep targets (self-referential). Level
sweep latches are per (level name, completed-period key): a level re-arms
when its period refreshes. Persistence capability (R1, capability-only)
covers the schema kinds EQH/EQL/PDH/PDL/SESSION_H/SESSION_L; PWH/PWL and
running extremes are **state-only** (the frozen `levels.kind` vocabulary has
no PWH/PWL — channels precedent).

## D12.4 — Sweep detection (§4.4, pinned)

High-side sweep of target price `P`: `high > P` AND `close < P` (both
strict) AND (`RVOL ≥ 1.5` **OR** `top_wick > 0.6 × (high − low)`), with
`top_wick = high − max(o, c)`; zero-range candles never sweep. Low-side
mirrored (`bottom_wick = min(o, c) − low`). **RVOL placeholder evaluates
`False`** (flagged, until the Volume Engine lands): with OR semantics a True
placeholder would make every wick-through a sweep — False conservatively
rests detection on the wick criterion alone; the swap re-runs determinism.

## D12.5 — Sweep + shift (§4.4, pinned)

A sweep is tagged "sweep + shift" when the frozen Structure Engine emits a
**CHOCH within the next 3 closed candles** (bars sweep+1 … sweep+3; the
sweep bar itself does not count). CHOCH events are *fed in* from the same
stream's ChochDetector — the Liquidity Engine is a pure consumer of
structure outputs and re-implements nothing.

## D12.6 — Premium/discount (§4.4 + A8)

Range = the last confirmed **5m** H pivot price ↔ last confirmed 5m L pivot
price (A8 external structure — this is the first 5m-pivot consumer, so the
composition now instantiates the 5m PivotDetector + Labeler). Midpoint is
linear (`(hi + lo) / 2`; §4.4 mandates no log transform here). Per closed 1m
candle: `close > mid` → premium, `close ≤ mid` → discount (boundary pinned
to discount), `None` until both 5m pivots exist.

## D12.7 — Cadence (composition pin)

1m closed candle: frozen structure chain first (…`choch.update`) →
`liq.on_choch(event if any)` → `liq.update(candle)` → payload. 5m closed
candle: 5m detector → labeler → `liq.on_external_pivot`. Same-bar CHOCH
therefore *precedes* `liq.update` yet never tags a same-bar sweep (D12.5).

## Freeze-audit addenda (recorded readings + fixes)

- **Partial-period suppression (D7 doctrine, freeze-audit fix):** a period
  (day/week/session) not observed from its exact boundary start is PARTIAL
  and is never promoted to a level — mirroring the builder's startup-candle
  discard. Gap contiguity inside an observed period is guaranteed by the
  P0.15 backfill; restarts re-seed partial periods that stay suppressed.
- **Out-of-order guard (freeze-audit fix, composition-level):** the
  reconnect path's accepted D7 residual can emit a stale bucket after its
  backfilled successors; `_StructurePipeline` drops non-increasing-ts
  candles with a WARNING before any engine ingests them.
- **Pool latch (freeze-audit fix):** latch key = (kind, member set); a pool
  whose members are a *subset* of a same-kind swept set stays latched
  (shrinkage is not fresh liquidity); a superset (gained member) is fresh
  per D12.2. Latches whose members left the pivot windows are pruned;
  level latches prune on promotion — both sets are bounded.
- **Accepted readings:** pool recompute is cross-kind (any new confirmed
  pivot re-evaluates both kinds' pools — deterministic, D12.2's sentence
  read as per-kind *clustering*, not per-kind *triggering*); pool-strength
  age is anchored at the pivot's CONFIRMATION bar (no-repaint convention).
