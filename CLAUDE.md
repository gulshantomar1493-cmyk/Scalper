# CLAUDE.md — Working Rules for MarketScalper

MarketScalper is a production-grade personal **market analysis & decision-support** platform.
It is NOT an automated trading platform, NOT an order-execution platform, NOT a broker terminal.
It generates trade recommendations; the owner always executes manually on their exchange and logs outcomes.
Owner: Gulshan. Solo-builder project.

## Frozen documents — never modify

- `docs/Architecture.md` — Blueprint **v1.2, FROZEN** (v1.2 = scope simplification: execution layer removed, engines untouched). Never redesign the architecture. No engines added or removed.
- `docs/IMPLEMENTATION_PLAN.md` — Implementation roadmap **v2.0, approved** (2026-07-14). Do not edit it without explicit owner instruction.

## Hard scope boundaries (v1.2 — never reintroduce)

No order placement · no trade/position management · no position sync · no broker auth · no order APIs · no kill switch/emergency flatten · no automated trailing/partials/time exits (management guidance is display-only text) · no PaperBroker/DeltaBroker/ExecutionProvider. Replay is for learning/validation/testing only — no replay execution.

## Process rules (non-negotiable)

1. Read all project files before doing anything.
2. Implement only approved modules — **one module (roadmap task) at a time**, in roadmap order.
3. Always explain the implementation plan and **wait for Gulshan's approval before coding**.
4. Never over-engineer. v1 scope: BTCUSDT + ETHUSDT only, 1m/5m, 3 strategies (S1/S2/S3), decision-support only. Single VPS, single process, single codebase; plain-Python provider interfaces wired in `main()` — no DI frameworks, no plugins, no microservices.
5. Always run tests after implementation.
6. Always update documentation after implementation.
7. `[DECISION]` roadmap tasks produce a short doc in `docs/decisions/` **before** their dependent code task. Decisions pin details the blueprint left open — they never change the architecture.
8. **Git milestone after every completed roadmap task** (rule set at P0.5): `git add .` → `git commit -m "P0.X: <short summary>"` → `git push`.

## Engineering disciplines (from Architecture §0 — locked)

1. **No repaint.** Every structural element computes only on confirmed closed candles. Store `ts` and `confirmed_ts`.
2. **Replay-first.** No engine is "done" until historical replay gives bit-identical output. Run the determinism harness after every engine change (same input candles → byte-identical output tables).
3. **No fake confidence %.** Score = weighted rule agreement (0.30 Structure / 0.30 Liquidity / 0.25 Volume / 0.15 Momentum), never probability. Two-tier display: Data Integrity + Signal Agreement.
4. **Validate before trust.** Minimum 200 logged recommendations with positive expectancy (after fees) before a strategy is marked TRUSTED. The platform never executes trades — there is no "live unlock".

## Stack (locked — see Architecture §2)

Python 3.12 + asyncio · FastAPI (WS + REST) · PostgreSQL 16 (append-only discipline) · Vanilla JS + TradingView Lightweight Charts v5 · FeedProvider abstraction with capability flags (BinanceFeed, ReplayFeed, optional DeltaFeed — public market data only; engines never see raw provider JSON) · single systemd process on a self-hosted Linux server (hosting provider = implementation decision, never named in docs) · pytest + pytest-asyncio.

## Repo layout

`backend/` providers, engines, recommendation layer & API · `frontend/` LWC v5 terminal · `database/` migrations · `deployment/` systemd/deploy · `scripts/` CI & tooling (determinism + import-boundary gates) · `tests/` pytest · `docs/` frozen docs + decisions.

## Conventions fixed at P0.2 (final)

Package name: `marketscalper` (stable, never rename) · deps in `pyproject.toml` use minimum compatible versions (`>=`), no strict pins unless absolutely necessary · config order (never change): `backend/config.example.yaml` → `backend/config.yaml` (git-ignored) → env vars override YAML · no secrets in git, ever · all logging via `logging.getLogger(__name__)`, UTC timestamps · `main()` in `marketscalper/main.py` is the only composition point · the project's single HTTP client is **aiohttp** (fixed at P0.11): used directly at call sites — no HTTP abstraction layers, no wrappers, no networking utilities.
