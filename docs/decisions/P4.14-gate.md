## The roadmap gate

P4.14: "2-week forward run; ≥60 recommendations; evaluator outcome
recorded for all; manual log complete for all taken; docs updated."

This is fundamentally a **live forward-run campaign** — it requires real
market data flowing for two weeks, ≥60 genuinely-generated
recommendations, and the owner manually logging every taken trade. It
cannot be executed by an autonomous build session; it needs real calendar
time and live operation. It belongs to the same deferred class as P1.22
(manual 50-line audit), the P2.24 rule-quality residual, and the P3.21
rule-quality residual — validation that is not automatable.

Per the established methodology, this record splits the gate into the part
that IS automatable (the machinery is wired and correct) and the part that
is not (the live campaign itself).

## Machinery readiness — PASSED (automated)

`tests/test_p4_gate.py` drives a single recommendation through the ENTIRE
Phase-4 chain against the real schema, proving every seam the owner's live
run depends on is wired and correct:

1. **Admission** — the §6→§7 flow admits an A_PLUS recommendation (D21.2).
2. **Persistence** — the SignalRecorder writes the signal row, the
   recommendation row, and the journal auto-context seed (P3.18/P4.6), and
   writes the DB row id back for the quick-log form (P4.7).
3. **Lifecycle + evaluator** — the entry fills and reaches TP1; the
   RecommendationLifecycle emits an `evaluated` transition with the
   candle-geometry `eval_r` (P4.2/P4.3).
4. **Eval persistence** — the transition's status + `eval_*` land on the
   recommendation row in one transaction (P4.5).
5. **Manual log** — the owner's Taken/Win/actual-R/notes/tags are written
   through the journal manual path (P4.7/P4.8).
6. **Analytics** — `compute_analytics` reflects the recommendation in the
   hypothetical stats, the manual results, AND the system-vs-actual delta
   (the campaign's core comparison), per strategy and per session (P4.11).
7. **Daily snapshot** — `format_daily_summary` renders the day's line
   (P4.13).
8. **Journal tab** — `journal_list` surfaces it with the rule-trace and
   both outcomes (P4.12).

A second test confirms a SKIPPED recommendation appears in the
hypothetical stats but is excluded from the manual results and the
system-vs-actual comparison — the exact gap the live campaign exists to
measure. Both run in the normal pytest step.
