## Roadmap objective

P3.21: "90-day replay → recommendations generated; top-20/bottom-20
score-ordering review; docs updated." Following the P2.24 precedent, the
automatable parts are verified at 100% coverage by independent
recomputation (`tests/test_p3_gate.py`); the non-automatable part (a
human trader agreeing the high-scored setups are genuinely better trades)
is deferred to Final Pre-Release Validation.

## What the gate validates

- **Recommendations generated end-to-end.** The seeded `rec_dataset`
  replayed across both v1 symbols drives the complete P3 chain
  (S1–S3 strategies → qualification → D17 planner → D21.2 admission) and
  produces real recommendations — verified present (`≥ 2`, one per
  symbol), each an S1 LONG at score ≈ 83.5, net RR ≈ 5.47.
- **Score ordering makes sense**, as a checkable invariant set over
  EVERY qualification the composition emitted (1,170 results, 1,112
  scored across both symbols):
  - **Formula integrity**: `score == 0.30·Structure + 0.30·Liquidity +
    0.25·Volume + 0.15·Momentum` exactly (the frozen §6 weights), on
    every scored bar.
  - **Verdict partition**: any gate fail → `NO_SIGNAL` with null
    score/components; else the frozen thresholds (`≥85 A_PLUS`,
    `≥75 TRADEABLE`, else `BELOW_THRESHOLD`) via the engine's own
    `verdict_of`.
  - **Integrity + agreement**: `data_integrity == PASS` iff G1∧G2; the
    agreement string is exactly `"{aligned} of 14 rules aligned"`
    (14 evaluable items — the composition always attaches the Volume
    Engine, D21.3) or the gate-fail sentinel.
  - **Component dominance**: a bar whose four components each dominate a
    reference bar's never scores lower — the "ordering makes sense"
    property, checked against a reference chosen by component-sum
    (immune to a corrupted score).
  - **Top/bottom evidence ordering** (the automatable form of the
    roadmap's top-20/bottom-20 review): the top-20 scored bars separate
    cleanly from the bottom-20 (min-top score > max-bottom score) and
    carry strictly more aligned rules (≈11 vs ≈2 on the gate run).
- **Every generated recommendation is valid**: its bar cleared the
  TRADEABLE threshold, the plan geometry is sound (SL/entry/TP ordering,
  TP2 beyond TP1), net RR(TP1) ≥ 1.0, and TP1 ≥ 1R.
- **Verdict spectrum**: `NO_SIGNAL` (warming), `BELOW_THRESHOLD`
  (climbing), and `TRADEABLE` all present in the stream (score range
  12.5 → 83.5). `A_PLUS` is not reached by this dataset — recorded, not
  required (it needs near-maximal components across all four families).

## Non-vacuousness

Four self-tests corrupt a captured run and assert the verifiers FAIL:
a bumped score (formula), a wrong-threshold verdict (partition), a
below-floor net RR (recommendation validity), and a component-dominating
bar given a low score (dominance). All confirmed to catch their
corruption.

## Recorded residual

Rule-quality-versus-trader-judgment (does a human agree the top-scored
setups are the better trades) is not automatable and is deferred to Final
Pre-Release Validation, alongside P1.22 and the P2.24 rule-quality

CALIBRATION (are the thresholds/weights right for real markets) is P5.
