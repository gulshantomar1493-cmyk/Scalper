# CLAUDE.md вҖ” Working Rules for MarketScalper

MarketScalper is a production-grade personal **market analysis & decision-support** platform.
It is NOT an automated trading platform, NOT an order-execution platform, NOT a broker terminal.
It generates trade recommendations; the owner always executes manually on their exchange and logs outcomes.
Owner: Gulshan. Solo-builder project.

## Frozen documents вҖ” never modify

- `docs/Architecture.md` вҖ” Blueprint **v1.2, FROZEN** (v1.2 = scope simplification: execution layer removed, engines untouched). Never redesign the architecture. No engines added or removed.
- `docs/IMPLEMENTATION_PLAN.md` вҖ” Implementation roadmap **v2.0, approved** (2026-07-14). Do not edit it without explicit owner instruction.

## Hard scope boundaries (v1.2 вҖ” never reintroduce)

No order placement В· no trade/position management В· no position sync В· no broker auth В· no order APIs В· no kill switch/emergency flatten В· no automated trailing/partials/time exits (management guidance is display-only text) В· no PaperBroker/DeltaBroker/ExecutionProvider. Replay is for learning/validation/testing only вҖ” no replay execution.

## Process rules (non-negotiable)

1. Read all project files before doing anything.
2. Implement only approved modules вҖ” **one module (roadmap task) at a time**, in roadmap order.
3. Always explain the implementation plan and **wait for Gulshan's approval before coding**.
4. Never over-engineer. v1 scope: BTCUSDT + ETHUSDT only, 1m/5m, 3 strategies (S1/S2/S3), decision-support only. Single VPS, single process, single codebase; plain-Python provider interfaces wired in `main()` вҖ” no DI frameworks, no plugins, no microservices.
5. Always run tests after implementation.
6. Always update documentation after implementation.
7. `[DECISION]` roadmap tasks produce a short doc in `docs/decisions/` **before** their dependent code task. Decisions pin details the blueprint left open вҖ” they never change the architecture.
8. **Git milestone after every completed roadmap task** (rule set at P0.5): `git add .` вҶ’ `git commit -m "P0.X: <short summary>"` вҶ’ `git push`.

## Engineering disciplines (from Architecture В§0 вҖ” locked)

1. **No repaint.** Every structural element computes only on confirmed closed candles. Store `ts` and `confirmed_ts`.
2. **Replay-first.** No engine is "done" until historical replay gives bit-identical output. Run the determinism harness after every engine change (same input candles вҶ’ byte-identical output tables).
3. **No fake confidence %.** Score = weighted rule agreement (0.30 Structure / 0.30 Liquidity / 0.25 Volume / 0.15 Momentum), never probability. Two-tier display: Data Integrity + Signal Agreement.
4. **Validate before trust.** Minimum 200 logged recommendations with positive expectancy (after fees) before a strategy is marked TRUSTED. The platform never executes trades вҖ” there is no "live unlock".

## Stack (locked вҖ” see Architecture В§2)

Python 3.12 + asyncio В· FastAPI (WS + REST) В· PostgreSQL 16 (append-only discipline) В· Vanilla JS + TradingView Lightweight Charts v5 В· FeedProvider abstraction with capability flags (BinanceFeed, ReplayFeed, optional DeltaFeed вҖ” public market data only; engines never see raw provider JSON) В· single systemd process on a self-hosted Linux server (hosting provider = implementation decision, never named in docs) В· pytest + pytest-asyncio.

## Repo layout

`backend/` providers, engines, recommendation layer & API В· `frontend/` LWC v5 terminal В· `database/` migrations В· `deployment/` systemd/deploy В· `scripts/` CI & tooling (determinism + import-boundary gates) В· `tests/` pytest В· `docs/` frozen docs + decisions.

## Conventions fixed at P0.2 (final)

Package name: `marketscalper` (stable, never rename) В· deps in `pyproject.toml` use minimum compatible versions (`>=`), no strict pins unless absolutely necessary В· config order (never change): `backend/config.example.yaml` вҶ’ `backend/config.yaml` (git-ignored) вҶ’ env vars override YAML В· no secrets in git, ever В· all logging via `logging.getLogger(__name__)`, UTC timestamps В· `main()` in `marketscalper/main.py` is the only composition point В· the project's single HTTP client is **aiohttp** (fixed at P0.11): used directly at call sites вҖ” no HTTP abstraction layers, no wrappers, no networking utilities.
