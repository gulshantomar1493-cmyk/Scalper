# DECISION P0.11 — Candle authority (A1) + Clock sync (A12)

**Date:** 2026-07-14 · **Status:** Accepted · **Scope:** implementation decisions only — zero architecture change.
Both decisions formalize what Architecture v1.2 already implies (§1 dual streams, §4.1 gap-safe backfill, §6 G1, the P0.28 zero-mismatch gate); nothing new is designed.

---

## D5 — Candle authority (A1)

**Decision.** The same 1m candle reaches the system two ways; the hierarchy is fixed:

1. **Tick-built candles (Candle Builder, from aggTrades) are the runtime truth.** Engines, StateStore, and candle persistence consume only these.
2. **Closed `kline_1m` candles are the reconciliation reference and the backfill/bootstrap source.** They check the truth (P0.14) and fill gaps where ticks were never received (P0.15/P0.16) — a gap-filled candle is kline-sourced by necessity.
3. **Every mismatch is logged, never silently overwritten.** Comparison rule: `o/h/l/c` exact; volume-ish fields (`v`, `qv`, `n_trades`, `taker_buy_v`) within 0.1% relative tolerance (aggTrade-vs-kline aggregation timing differs legitimately at the margin). Log at WARNING with both rows; the tick-built row remains the persisted truth.

**Why.** Without a declared authority, P0.14 has no rule for which row survives, and the P0.28 gate ("zero candle mismatch vs official klines") is unmeasurable.
**Consumers.** P0.12 (builder = truth producer), P0.14 (comparison + logging exactly as above), P0.15/P0.16 (kline backfill/bootstrap), P0.28 (gate).

## D6 — Clock sync (A12)

**Decision.** "Clock sync < 2s" (§6 G1) is measured as:

- **Offset = Binance server time (`GET /api/v3/time`) minus the local wall-clock midpoint** of the request's send/receive instants (midpoint neutralizes network latency).
- **Sampling cadence: every 5 minutes**, as a background task inside the single process.
- **G1 fails when |offset| > 2.0s** — and also when the offset is **unknown**: no successful sample yet, or 3 consecutive sampling failures (an unverifiable clock is treated as an unsynchronized one).

**Why.** Every no-repaint guarantee hangs on candle-close timestamps; G1 must catch a drifted or unverifiable clock.
**Consumers.** P3.1/P3.2 (G1 reads `offset_s` / `in_sync`), the sampler implemented in this task (`providers/binance.py`).

## Dependency record (owner-approved at P0.11)

The project's single HTTP client is **aiohttp** (runtime dependency, minimum-version style). It is used **directly** where needed — no HTTP abstraction layer, no wrappers, no networking utilities — starting with the clock-offset sampler, and reused later for historical candle downloads (P0.15/P0.16), bootstrap, and any future REST operation (e.g. optional DeltaFeed public REST, P6). One client, consistently, project-wide.
