# DECISION P3.18 — Signal/recommendation persistence + Volume rubric (D21)

**Date:** 2026-07-19 · **Status:** Accepted (autonomous execution
directive; pins per the D9–D20 precedent) · **Scope:** implementation
decision only — zero architecture change. Sole specification for
roadmap P3.18 (rule-trace reasoning + persistence) and for the
pre-recorded resolution of the D16.3 Volume-component placeholder.

## D21.1 — Signal rows (immutable, append-only)

Every Signal a strategy emits persists as one `signals` row at its
emission bar, regardless of the qualification verdict — the row is the
forensic record "the system saw this setup under this exact state"
(§0 rule 4 needs the full evidence stream, and P4.11's system-vs-actual
analytics consume it). Columns: `ts` = the signal's `created_ts`,
`tf` = '1m', `strategy`/`direction` from the Signal, `score` = the
same-bar QualificationResult score (NULL in gate-fail/warming eras per
§6), `gates` = JSON {verdict, data_integrity, agreement, gates:[{name,
passed, flagged, detail}]}, `components` = the component dict (NULL on
gate fail), `state_snapshot` = the bar's complete structure payload
(the §8/A17 forensics dump — includes the signal's own facts),
`engine_version` = the D1 stamp.

## D21.2 — Recommendation rows and the §7/§8 flow

A recommendation is created iff ALL of:
1. the same-bar verdict is `TRADEABLE` or `A_PLUS` (§6 "Tradeable
   threshold" — a below-threshold or gate-failed bar recommends
   nothing);
2. the D17 planner accepts: `plan.status == 'suggested'` (geometry
   valid, net RR(TP1) ≥ 1.0 after fees);
3. `plan.rr_floor_ok` is True — this IS the G6 enforcement (D17
   recorded "owner decision at strategy time": the bar-level G6 gate in
   the frozen QualificationEngine STAYS flagged-pass because a plan
   exists only per-signal after qualification runs; the floor —
   net RR(TP1) ≥ 1.0 AND net RR(TP2) ≥ 1.5 when TP2 exists — is
   enforced here, at creation, where it is computable).

Row: `signal_id` FK, `ts` = signal ts, plan fields (`entry_px`, `sl`,
`tp1`, `tp2`, `suggested_qty`, `risk_amt`,
`est_fees` = qty × fee_per_unit, `net_rr_tp1`), `status` = 'active'
(lifecycle transitions are P4.2's). Planner-rejected or
below-threshold signals produce NO recommendation (D20.1) — the signal
row still exists.

§8 reasoning = the deterministic rule-trace already carried by the
payload (signal `facts` + qualification `reasons` + the plan numbers);
P3.18 adds no free text. `journal.reason_text` is seeded at P4.6, not
here; the signal row's `state_snapshot` already lets any consumer
recompute the §8 card verbatim.

## D21.3 — Volume component rubric (the D16.3 placeholder resolved)

D16.3 recorded Volume = 0 flagged "until the Volume Engine lands",
citing the D11/D12 RVOL-placeholder doctrine whose resolution pattern
(P2.2) is an optional seam that preserves legacy behavior byte-for-byte
when absent. The Volume Engine is frozen; this decision resolves the
placeholder the same way. §6 gives no Volume rubric (its rubric block
is "e.g. Liquidity") — the items are pinned here from the frozen §4.6
outputs, mirroring the +40/+30/+20/+10 shape and the D16.3 conventions
(trend as the direction-neutral reference; None fails its item; all
constants uncalibrated, P5-owned):

- **participation +40**: `rvol ≥ 1.5` inclusive (None → 0; the D11/D12
  threshold family).
- **delta alignment +30**: sign of the bar's `delta` agrees with the
  trend (BULLISH ↔ positive, BEARISH ↔ negative); zero delta,
  RANGE/None trend → 0 (the "nothing to align" D16.3 precedent).
- **VWAP side +20**: close on the trend side of `session_vwap`
  (BULLISH: c > vwap, BEARISH: c < vwap); vwap None or RANGE/None
  trend → 0.
- **no reversal warning +10**: neither `absorption` nor `exhaustion`
  is set this bar (the "no opposing CHOCH" absence pattern).

Seam: `QualificationEngine(..., volume=None)` — a frozen VolumeEngine
reference (its own constructor's trend/ATR/momentum/regime precedent).
`volume=None` → component 0 flagged and m = 10, byte-identical to the
frozen legacy (every pre-existing test green unchanged). Attached →
the four items are evaluable: m = 14, the component is real, flag
cleared. Composition attaches it (cadence already valid: volume.update
runs in phase 1, volume.classify before confluence/qualification per
D19.8). The Liquidity R-item stays flagged (bar-level qualification
still has no plan; recorded, P5-era). Max achievable score becomes
0.30×100 + 0.30×90 + 0.25×100 + 0.15×100 = 97.0 — TRADEABLE/A_PLUS
reachable, as D16.3's consequence note anticipated.

G3 (session filter) stays flagged-pass: neither §5 nor D20 defines
per-strategy session sets; owner calibration item (P5-era), recorded.

## D21.4 — engine_version stamp (the D1 obligation lands)

Each engine module now declares `ENGINE_VERSION = 1` (momentum,
structure, trendline, liquidity, orderblock, fvg, volume, confluence,
qualification, risk, strategy). The stamp
`<git-short-hash>+<engine>=<n>;...` is assembled once at composition
startup (`git rev-parse --short HEAD`, fallback `nogit`) and written on
every signal row. Adding a declared constant to frozen modules is the
D1-mandated bookkeeping, not a logic change; any future logic change to
a frozen engine must bump its constant per D1.

## D21.5 — Equity source

`plan_trade` requires equity > 0 (D17 geometry check). Pin:
`MARKETSCALPER_EQUITY_USD` env var, default **10000.0** — a recorded
display-scaling default (qty/risk_amt are §7 display-only suggestions;
net RR is equity-independent, so recommendation admission never depends
on it). `deployment/env.example` gains the key. Replay/test wiring uses
the same default constant — deterministic, replay ≡ live.

## D21.6 — Recorder architecture (composition, not engine)

The pipeline computes everything pure (plan + recommendation admission
are deterministic functions of frozen outputs — they run in EVERY
wiring, so payloads are identical with or without persistence). A
`SignalRecorder` (constructed only in live `main()` with the pool and
the stamp) receives each bar's records after `step()` and awaits the
inserts on the bus dispatch path — the CandleWriter doctrine verbatim:
failures caught, logged, counted; the pipeline never dies on a DB
error; recommendation insert is skipped when its signal insert failed
(FK discipline). Replay sessions and tests pass no recorder — zero
rows, the R1/F2 replay-never-writes discipline. Restart duplicates are
possible in the append-only forensics stream (no dedup by design — the
D5 duplicate-candle precedent).

## D21.7 — Payload

Additive key `recommendations`: the last 10 admitted recommendations
per symbol (the `signals` deque precedent) — each entry carries the
signal identity (strategy, direction, created_ts), the §7 plan numbers
(entry/sl/tp1/tp2/qty/risk_amt/est_fees/net_rr_tp1/net_rr_tp2), the §7
management-guidance lines (display-only text), and the admitting
score/verdict. The frontend ignores it until P3.19. Determinism: V1/V2
streams carry it empty (V2's gate shapes never signal; V1 never
shifts); V3's S1 fires at score 52.5 (BELOW_THRESHOLD — see D21.8), so
its recommendations key is present-but-empty. A TRADEABLE +
recommendation-carrying hashed stream is P3.20's milestone, using the
existing `seed_candles` surface to warm rvol.

## D21.8 — Consequences recorded

- An S1 signal bar structurally forfeits the Structure "no opposing
  CHOCH" +20 (its own reversal CHOCH opposes the prevailing trend), so
  its Structure component ceils at 80.
- Unseeded runs (no 20-day RVOL history) lose ONLY the Volume
  participation +40 item (rvol None); the other three Volume items are
  seeding-independent — delta alignment +30, VWAP side +20 (session
  VWAP builds from the stream), and the no-absorption/exhaustion +10
  (unconditionally granted unseeded, since both warnings require a
  spike and a spike requires rvol). Unseeded Volume therefore ceils at
  60, not 0/30. Consequence corrected after the freeze audit: an
  unseeded S1 box CAN reach TRADEABLE if delta and VWAP both align
  (ceiling 0.30×80 + 0.30×90 + 0.25×60 + 0.15×100 = 81.0). Whether any
  given bar recommends is data-dependent; the V3 dataset happens to
  land at score 52.5 (Volume 30 = VWAP +20 + no-warning +10, its
  synthetic 50/50 taker split zeroing delta; Liquidity 50 = shift +30 +
  zone +20, ASIA_L being a key level not a +40 pool), which is why V3's
  recommendations stream is empty. Seeded runs unlock the +40 item and
  the +30 delta item once real taker volumes flow (P3.20).
- `update_recommendation_status` / `update_recommendation_eval`
  remain unused until P4.2/P4.3 (their owners).
