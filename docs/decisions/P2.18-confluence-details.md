## D15.2 — Geometry and counting

- Anchors = the tradeable **zone** objects only: active OBs, active breakers,
  unfilled FVGs (§5 entries are at OB/FVG zones; lines/points are evidence,
  not entries).
- Every object projects to a closed price interval (table above). Interval
  distance `d(A,B)` = 0 if they intersect, else the gap between nearest
  edges. Two objects stack iff `d ≤ 0.3 × ATR` — **inclusive** ("within",
  the D14 boundary convention), ATR = the current 1m ATR at the evaluation
  bar. ATR unwarm → no confluence output at all (refuse-until-ready).
- `zone_quality(anchor)` = 1 (the anchor itself) + count of **other** stack
  objects within tolerance of the anchor's interval. Each object counts
  once; the anchor never counts other anchors' members for them (each
  anchor is scored independently). `htf_magnet` = `zone_quality ≥ 3`
  (§4.5: "3+ overlapping objects").
- Member tags recorded per anchor for display/reasoning: the anchor's own
  kind plus each stacked object's tag — `OB`, `BREAKER`, `FVG`,
  `TRENDLINE`, `EQH`, `EQL`, or the key-level name (`PDH`, `LONDON_H`, …).
- Deterministic output order: `zone_quality` desc, then anchor
  `created_ts` desc (newer first), then kind (`BREAKER` < `FVG` < `OB`
  lexicographic), then direction. Total order — no ties survive because
  `created_ts` is unique per (kind, direction) stream.

## D15.3 — Shape, composition, persistence

- **Pure function, no state** (the simplest honest §0 form):
  `confluence_zones(...) -> list[ConfluenceZone]` in `engines/confluence.py`,
  recomputed per closed candle from frozen-engine state — exactly like
  `TrendlineDetector.candidates()`. `ConfluenceZone` is a frozen dataclass:
  kind, direction, lo, hi, count, members (tuple), htf_magnet, created_ts.
- Cadence: computed in the composition step after `fvg.update`, before the
  Qualification Engine consumes it (D16), before the payload.
- Payload gains `"confluence"`: the zone list projected verbatim, display-
  capped to the top 10 (`_ZONES_SHOWN = 10`, the `_EVENTS_SHOWN`
  precedent); the Qualification Engine receives the **uncapped** list.
- **No persistence.** Neither §3 nor the roadmap defines a confluence row;
  zone_quality is a scoring input (§4.5) and signals rows are P3.18's.

## Freeze-audit addenda (2026-07-18 — behavior-identical readings, recorded)

- **Sort totality correction:** the D15.2 claim "no ties survive" is not
  upheld by the key alone — one break candle can spawn two same-direction
  breakers with identical `created_ts` (the frozen OB engine stamps the
  break bar), and equal counts then tie all four key components. Behavior
  stays deterministic: Python's stable sort falls back to the anchor
  construction order (blocks → breakers → gaps, each in engine tracking
  order), which is deterministic per stream. The fallback is part of the
  pinned order.
