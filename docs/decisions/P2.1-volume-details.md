**Owner rulings on the open questions:** (1) `BREAK_RVOL_MIN = 1.5`

## D19.1 — RVOL (§4.6 verbatim, boundaries pinned)

- `rvol[i] = v[i] / median(v at the same minute-of-day, last 20 days)`.
  Crypto 24×7 → **1440 UTC minute-of-day buckets** per symbol.
- Each bucket is a ring of the **last 20 observed** volumes for that
  minute-of-day (a data gap does not poison the bucket — the window
  slides over observed values; it just reaches 20 via older days).
- Median = the P1.4 convention: sort, middle element; even count never
  occurs (window is exactly 20 → mean of the two middles).
- **Warm-up (D7):** a bucket computes only with a FULL window (20
  observations). Fewer → `rvol = None` for candles in that bucket — a
  smaller sample is a different statistic; never fabricated.
- **No self-reference:** candle *i*'s rvol is computed against the bucket
  state BEFORE folding candle *i*; the fold happens after.
- Degenerate guard: `median ≤ 0` → `rvol = None`.
- Only closed **1m** candles feed buckets (post-guard composition stream;
  5m candles are ignored engine-wide — folding both would double-count).

## D19.2 — Warm-up seeding, cache, replay behavior

- The engine is **pool-less and pure** (R1 preserved): it exposes
  `seed(candles)` — a plain chronological fold of historical 1m candles
  into the buckets (buckets only; VWAP/delta/session state is NEVER
  seeded — those are day-anchored and rebuild from the live stream).
- **Seeding rule (pinned):** at stream start, composition performs one
  storage read per symbol — the 20 days immediately preceding the stream
  start — and calls `seed()`. The SAME rule applies to live startup and
  to F2 replay sessions (seed window = 20 days before the replay range
  start). Seeded state is a pure function of (DB content, window) →
  restart-deterministic and replay-identical.
- Unseeded operation (tests, harness, empty DB) is fully supported and
  deterministic: buckets fill from the stream; rvol stays None until a
  bucket completes (≈20 days live). Nothing else degrades.
- Cache strategy: the 1440×20 ring buffers ARE the incremental cache
  (per symbol ≈ 28,800 floats — trivial, bounded, O(1) per candle plus
  an O(20 · log 20)-equivalent median per candle; no persistence, no
  recomputation from history at runtime).

## D19.3 — Session VWAP + bands (§4.6)

- **Session = the UTC day** (00:00 UTC reset — §4.6's crypto anchor).
  This is deliberately NOT the A9 four-session map; pinned to avoid
  conflation.
- `typical_price = (h + l + c) / 3` (pinned — §4.6 names but does not
  define it). `vwap = Σ(tp·v) / Σ(v)` over the day's closed 1m candles,
  cumulative per candle. `Σv = 0` so far → `vwap = None`.
- **σ (pinned formula):** volume-weighted deviation vs the CURRENT vwap
  via the three running sums `Σv, Σ(tp·v), Σ(tp²·v)`:
  `var = max(0, Σ(tp²·v)/Σv − vwap²)`, `σ = sqrt(var)` (the max(0,·)
  clamps float rounding; precision at BTC scale verified adequate —
  ~10 significant digits of margin). Bands = `vwap ± 1σ`, `vwap ± 2σ`;
  σ = 0 early in the day is valid, not an error.
- Reset is **candle-ts driven** (first candle whose UTC date differs) —
  no wall clock (determinism; the D12 rollover precedent).
- **D7 day-continuity:** the day's vwap/bands (and cum_delta, D19.5) are
  emitted only while the day is *completely observed*: accumulation began
  at the day's first observed candle being 00:00, and every subsequent
  candle was minute-contiguous. P0.15 backfill publishes missed candles
  in-stream before live resumes, so reconnect gaps do not break
  continuity; a process restart mid-day does → the day's values are None
  until the next 00:00 reset (absent, never wrong).

## D19.4 — Anchored VWAP (§4.6, A8)

- **Anchor = the most recent CONFIRMED 5m pivot** (A8 external
  structure), either kind; if an outside bar confirms both H and L, the
  last one delivered in the pinned pipeline order wins. No confirmed 5m
  pivot yet → `anchored_vwap = None`.
- Accumulation covers 1m candles with `ts ≥ anchor.ts` (the pivot's 5m
  window start). Since confirmation arrives k=2 windows later, the
  engine keeps a rolling buffer of the last **1440** 1m candles (24 h);
  on each new confirmed anchor it recomputes the sums from the buffer
  (rare event, bounded cost, deterministic) and folds incrementally
  thereafter.
- D7: anchor older than the buffer, or the buffer not minute-contiguous
  across [anchor, now] (e.g. post-restart) → None until the next anchor.
- Value only — §4.6 defines bands for the session VWAP alone.

## D19.5 — Delta and cumulative delta (§4.6 verbatim)

- `delta[i] = taker_buy_v − (v − taker_buy_v)` (= 2·taker_buy_v − v).
  Pure per-candle arithmetic — always valid, no warm-up.
- `cum_delta` per session = the SAME UTC-day session as D19.3, same
  reset, same D7 day-continuity rule (restart mid-day → None until the
  next reset).

## D19.6 — Absorption (§4.6; thresholds pinned, no new magic numbers where a frozen constant exists)

A per-bar event, all four conditions on the same closed 1m candle:
1. **High volume:** `rvol ≥ 2.0` (the §4.6 spike constant — one notion
   of "high volume"); `rvol None` → no absorption (D7).
2. **High |delta|:** `|delta| ≥ 0.5 × v` (taker share ≥ 75 % one-sided).
   The 0.5 ratio is **uncalibrated, P5-owned** (D9 doctrine).
3. **Small range:** `(h − l) < 0.5 × ATR` strict (§4.6 literal). ATR
   unwarm → no absorption.
4. **At a key level:** the candle's range `[l, h]` comes within
   `0.3 × ATR` (the frozen D15 band constant, reused) of any promoted
   liquidity key level or any pool mean price — consumed from the frozen
   Liquidity Engine's outputs, passed in by composition, never
   recomputed.
Output: `AbsorptionEvent(level_tag, delta_sign)` — the §4.6 "reversal
warning at that level" display datum. Zero-range candles are eligible
(range 0 < threshold: legitimate high-volume standstill).

## D19.7 — Spike and exhaustion (§4.6)

- **Spike:** `rvol ≥ 2.0` **inclusive** (§4.6 literal "≥"). None → False.
- **Exhaustion:** spike AND long wick AND at the day's range extreme:
  - long wick: dominant-side wick fraction `> 0.6` of range (reusing the
    frozen D12 sweep-wick constant; zero-range candle → no exhaustion);
  - range extreme: the candle sets-or-touches the running day extreme —
    `h ≥ day_high_so_far` → `exhaustion = 'TOP'` (upper wick), mirror
    `'BOTTOM'`; day extremes consumed from the frozen Liquidity Engine's
    `running_extremes`, passed in by composition.

## D19.8 — Cadence, inputs (two-phase update, pinned)

- **Phase 1 — `update(candle)`**, placed immediately after
  `momentum.update` (ATR is warm for the bar; needs nothing else):
  computes rvol, session vwap/σ/bands, anchored VWAP, delta, cum_delta,
  spike. **This makes the bar's rvol available BEFORE the trendline
  book, liquidity, and every later engine run** — the property the P2.2
  consumers require.
- **Phase 2 — `classify(candle, key_levels, pools, extremes)`**, placed
  immediately AFTER `liq.update`: computes absorption + exhaustion from
  liquidity's post-update state. (Two-phase intake follows the liquidity
  `on_choch`/`update` precedent — no new pattern.)
- 5m candles: never delivered to this engine.
- Exact inputs: closed 1m `Candle`; the shared `IncrementalATR` by
  read-only reference; phase-2 args (`key_levels: dict`,
  `pools: list[LiquidityPool]`, `extremes: dict`) passed per bar by
  composition; `on_anchor(pivot)` — confirmed 5m pivots delivered in
  step_5m alongside the liquidity external-pivot feed (the D19.4 anchor
  source, listed here for completeness). `seed(candles)` before the
  stream (optional).

## D19.9 — Outputs (consumer contract)

Read-only properties, valid after the respective phase for the bar:
`rvol: float|None` · `session_vwap: float|None` ·
`band_1_up/band_1_dn/band_2_up/band_2_dn: float|None` (None iff vwap
None) · `anchored_vwap: float|None` + `anchor_ts: datetime|None` ·
`delta: float` · `cum_delta: float|None` · `spike: bool` ·
`absorption: AbsorptionEvent|None` · `exhaustion: 'TOP'|'BOTTOM'|None`.
Consumers this milestone: composition payload (`volume` subdict — shape
finalized at implementation) and the two P2.2 seams. The D15
VWAP-band stack member and the D16.3 qualification Volume rubric remain
OUT of scope — separate owner decisions after this engine ships.

## D19.10 — P2.2 placeholder replacement rules

- **Seam-only changes** to the two frozen modules, exactly at the
  recorded flags — pre-authorized by roadmap P2.2 + D11.9 + D12.4:
  - `trendline.py` break-watch qualification AND-arm: real rule
    `rvol ≥ 1.5` inclusive (**proposed** — see open questions), captured
    at the through-bar. `rvol is None` → the arm passes (the recorded
    D11 legacy direction — unwarm RVOL must not suppress every BREAK).
  - `liquidity.py` sweep OR-arm: real rule `rvol ≥ 1.5` inclusive
    (**proposed**). `rvol is None` → arm False (the recorded D12 legacy
    direction — unwarm RVOL must not make every wick-through a sweep).
- Mechanism: each module receives the bar's rvol through an optional
  provider seam set at composition; provider absent → byte-identical
  legacy placeholder behavior (all existing tests and records hold).
- **Replay-identity property (deliberate):** with `rvol = None` the
  swapped arms reproduce the placeholder behavior EXACTLY — so the
  unseeded V1 determinism harness and every V1-pinned test value remain
  valid unchanged. The swap alters seeded/warm behavior only. P1
  determinism re-run is still mandatory (roadmap P2.2).

## D19.11 — Determinism, replay identity, D7 summary

Pure fold of (seed, candle stream, ATR reads, phase-2 args): no clock,
no randomness, no I/O. Live ≡ replay whenever seeds are equal, and the
D19.2 seeding rule makes them equal by construction. Every
insufficient-history case yields None/False (rvol short bucket, vwap and
cum_delta broken day, anchored VWAP without anchor/coverage, absorption
and exhaustion without warm inputs) — absent, never fabricated (D7).

## D19.12 — Edge cases (pinned)

Zero-volume candle: folds into its bucket (a real observation); may
drive `median → 0` → rvol None guard. Zero-range candle: delta/rvol
valid; absorption eligible; exhaustion excluded. Day-boundary candle
23:59 belongs to its day; 00:00 starts the new one. UTC has no DST. The
first candle ever: delta valid, everything else warming. Duplicate or
out-of-order candles never reach the engine (composition guard).
Restart: buckets re-seed from storage (identical result), day-anchored
state stays None until the next boundary (D19.3).
