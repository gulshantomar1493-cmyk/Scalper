# DECISION D24 — P3.3 resolution: G3 session filter + G4 news blackout

**Date:** 2026-07-19 · **Status:** Accepted (final-audit remediation) ·
**Scope:** implementation decision only — resolves the two §6 hard gates
(G3, G4) that D16.2 recorded as flagged placeholders whose deferral
targets (the strategies for G3; an owner-owned events.yaml for G4) closed
without absorbing the work. Sole specification for the P3.3 residual.

The final release audit found G3 and G4 were the only two gate INTENTS
delivered nowhere (G5 became real at P4.9/D23; G6's RR floor is enforced
by the D17 planner). This decision closes both — G3 with a real gate, G4
with a documented owner-operational deferral.

## D24.1 — G3 session filter (implemented)

§6 G3 is "Session filter: strategy-allowed session only". D16.2 routed
the per-strategy session sets to S1–S3 (P3.12+), but the frozen strategy
engine (D20) implements no session gating. Rather than reopen the frozen
strategies, G3 is re-homed to the QualificationEngine where §6 places it —
a real, deterministic hard gate driven by the candle's own timestamp (the
D6-clock-seam / P4.9-psych-seam precedent, but deterministic, so it runs
in replay too):

- **Policy (A9 default):** the LATE session (21:00–00:00 UTC, the frozen
  D12.1 map's low-liquidity window) is excluded — no strategy trades it.
  ASIA / LONDON / NY pass. `G3.passed = session_of(candle.ts.hour) !=
  "LATE"`, `flagged=False` (a real gate now). A G3 fail → NO_SIGNAL (§6),
  so during LATE the platform recommends nothing.
- **Determinism:** G3 is a pure function of `candle.ts` — replay ≡ live.
  Verified the V1–V4 determinism datasets span only ASIA/LONDON/NY (never
  LATE), so their hashes are unchanged; the always-on gate is nonetheless
  a real logic change, so `qualification.ENGINE_VERSION` bumps 1 → 2 (the
  first D1 bump — the discipline in action).
- **Not per-strategy:** a single account-level session policy (LATE
  excluded) is the A9 default. Per-strategy session sets remain a P5-era
  calibration item (uncalibrated, the D9 doctrine) — recorded, not built.

## D24.2 — G4 news blackout (owner-operational, deferred)

§6 G4 is "News blackout: ±5 min of high-impact scheduled events". It
requires two things this build cannot supply autonomously:

1. **The A11 events.yaml format** — a [DECISION] P3.1 explicitly recorded
   as "owner-owned" (the owner names the calendar schema and the
   high-impact event set for their markets). This is a product decision.
2. **An external events calendar** — real scheduled-event data (FOMC/CPI
   etc.), an external dependency the platform does not have.

G4 is therefore re-classified as OWNER-OPERATIONAL and stays a flagged
placeholder pass ("no events calendar yet") until the owner pins the A11
format and supplies the calendar — the same legitimate-deferral class as
the live-campaign gates (P4.14/P5.8) and the manual rule-quality audits
(P1.22). When the format lands, G4 becomes a real gate the same way G3
did (a deterministic check of `candle.ts` against the loaded event
windows). Recorded here so the residual is tracked, not silently dropped.

## D24.3 — Consequences

- G3 real: recommendations are suppressed during the LATE session
  (21:00–00:00 UTC). Existing tests at 09:00 (LONDON) / gate hours 3/9/15
  are unaffected; the G3-flagged-placeholder assertion is updated to the
  real gate, and a LATE-session fail case is added.
- `qualification.ENGINE_VERSION = 2` — the D1 stamp now reads
  `…;qualification=2;…` on every persisted signal, distinguishing
  pre/post-G3 signals forensically.
- G4 remains flagged; the payload still marks it provisional, so any
  verdict during a (future) news window reads as un-vetted until the
  owner wires the calendar.
