# DECISION P2.16 — Fair Value Gap Engine detail set (§4.5 pins)

**Date:** 2026-07-18 · **Status:** Accepted (engine-wise directive; pins per
the D9–D13 precedent) · **Scope:** implementation decision only — zero
architecture change. Sole specification for the FVG Engine (roadmap P2.16).

---

## D14.1 — Detection (§4.5 verbatim, boundaries pinned)

Evaluated on every closed 1m candle as candle-3 of the rolling 3-candle
window: **bullish FVG** iff `candle1.high < candle3.low` (strict) → gap
`[candle1.high, candle3.low]`; **bearish FVG** iff `candle1.low >
candle3.high` (strict) → gap `[candle3.high, candle1.low]`. At most one
FVG per bar (both directions simultaneously is arithmetically impossible).
Identity = (direction, candle3 ts). **Noise filter:** gap size
`(hi − lo) ≥ 0.3 × ATR(14,1m)` at the detection bar — "minimum gap size"
reads as *at least*, so the boundary is **inclusive** (unlike the
strict-`>` displacement family; recorded deliberately). ATR unwarm → no
detection (refuse-until-ready).

## D14.2 — Lifecycle (§4.5: 50% = "CE tested", full fill = archived)

`CE = (lo + hi) / 2`. Per closed candle, existing gaps only (a gap is
never evaluated on its creation bar — the D13 convention; candle3's own
extreme merely borders the gap):
- **Bullish gap** (price above, fills from above): full fill when
  `low ≤ lo` (inclusive — the far edge was reached) → status `filled`,
  archived (dropped from tracking, the OB-freeze precedent); else CE test
  when `low ≤ CE` (inclusive) → status `ce_tested` (sticky). A single
  candle can go active → filled directly.
- **Bearish gap** mirrored: `high ≥ hi` → filled; else `high ≥ CE` →
  `ce_tested`.
- **"Invalidated" is not implemented** — §4.5 defines no invalidation for
  FVGs (owner conditional: "if defined by the spec"; it is not).

## D14.3 — Bounds, persistence, cadence

- Track the most recent **10 unfilled** gaps per direction (D13 precedent;
  explicitly arbitrary, P5-owned).
- Persistence capability-only (R1): kinds `FVG_BULL`/`FVG_BEAR`,
  `p1 = hi`, `p2 = lo` (§3 top/bottom), statuses
  `active`/`ce_tested`/`filled` — `ce_tested`/`filled` are documented
  app-layer vocabulary additions (no CHECK constraints; the D11
  `'archived'` precedent).
- Cadence: after the Order Block engine in the composition step
  (…`ob.update` → `fvg.update(candle)` → payload). The engine holds its
  own 3-candle window; pure consumer of closed candles + the frozen ATR.

## Freeze-audit addenda (2026-07-18 — behavior-identical readings, recorded)

- **Trim removal mechanism:** `_trim` removes stale gaps via `list.remove`
  (dataclass equality). Correct because no two tracked gaps can ever be
  field-equal: `created_index` is a strictly monotone per-engine counter
  and at most one gap exists per bar (D14.1). Same mechanism as the frozen
  Order Block engine. The uniqueness invariant is load-bearing.
- **Creation-bar exemption is structurally vacuous for FVGs** (formally
  proved in the freeze audit): a bull gap's `hi` IS candle3's low (mirror
  for bear), so the creation candle borders the gap and can never fill or
  CE-test it under any evaluation order. The D13-convention ordering is
  kept as belt-and-suspenders; unlike OBs, it is not observable here.
- **Strict detection at equality:** the equality → no-gap outcome is
  test-pinned; the strict-`<` mechanism is only distinguishable from `<=`
  at ATR exactly 0.0 (an all-flat degenerate market), where a `<=` variant
  would create a zero-size gap. Not separately tested; the outcome
  contract is what D14.1 mandates.
