# DECISION P1.7 — RANGE formula (A6) + external structure (A8)

**Date:** 2026-07-18 · **Status:** Accepted · **Scope:** implementation decision only — zero architecture change.
Resolves A6 and A8 (roadmap Part B) exactly as far as P1.8 (trend-state machine) requires; nothing beyond that is frozen here.

---

## Objective

§4.2 defines the trend states as `BULLISH = HH + HL`, `BEARISH = LH + LL`, `RANGE = alternating / overlap > 60% of last 20-bar range` — the RANGE clause is not implementable as written, and "external structure" is referenced by three later consumers without a definition. This decision freezes both so P1.8 can be a pure transcription.

## D10 — Decision summary

1. **RANGE (A6)** adopts the roadmap Part B resolution: *alternating (mixed) labels OR ≥60% of the last 20 closed candles' bodies inside the last swing band* — with every term pinned below.
2. **External structure (A8)**: external swings **are the 5m-confirmed pivots** (k=2, the existing P1.5 detector on the 5m stream); 1m pivots are internal structure. One definition; consumers: premium/discount (P2.12), anchored VWAP (P2.4), S1 TP2 (P3.13). The trend-state machine itself is **timeframe-generic** — instantiated per `(symbol, tf)`, consuming that stream's own pivots, labels, and candles (S2's 5m CONTEXT reads the 5m instance; the A8 "external" designation matters only to the three consumers listed).

## Final formulas (total, deterministic, evaluated once per closed candle, after that candle's pivot detection/labeling)

Inputs at evaluation time, all from the same `(symbol, tf)` stream: `H` = last confirmed H pivot, `L` = last confirmed L pivot (labeled per P1.6), and the last 20 closed candles.

```
1. if H or L is missing, or H.label is None, or L.label is None:
       state = None                                  (unknown — warm-up)
2. elif band_warm and inside_count >= 12:            (band test wins ties)
       state = RANGE
3. elif H.label == 'HH' and L.label == 'HL':
       state = BULLISH
4. elif H.label == 'LH' and L.label == 'LL':
       state = BEARISH
5. else:                                             (mixed labels)
       state = RANGE
```

where:

```
band_lo, band_hi = min(H.price, L.price), max(H.price, L.price)
band_warm        = at least 20 closed candles have been observed
body of candle   = [min(o, c), max(o, c)]
body inside band = min(o, c) >= band_lo AND max(o, c) <= band_hi   (inclusive)
inside_count     = number of the last 20 closed candles whose body is inside
```

The classification is **memoryless**: a pure function of the current inputs, no hysteresis, no sticky state — the simplest deterministic form, and the one replay can reproduce bit-identically.

## Exact comparison rules

- Label matching is exact string equality on the P1.6 labels; seed pivots (`label None`) block classification (rule 1), they never count as "alternating".
- `inside_count >= 12` uses **≥** (12 of 20 = exactly 60% qualifies), per the Part B wording "≥60%".
- Band containment is **inclusive** on both edges: a body endpoint exactly on `band_lo`/`band_hi` counts as inside.
- The band uses pivot **prices** only (`min`/`max` normalizes the degenerate case where the last L price exceeds the last H price after a violent move); labels are irrelevant to the band.
- Wicks are irrelevant: bodies only, per the Part B formula.

## Boundary behaviour (worked)

- `inside_count = 12` → RANGE (even if labels read HH∧HL). `inside_count = 11` → falls through to the label rules.
- Labels HH∧LL or LH∧HL (mixed) → RANGE via rule 5.
- Body with `o == c` (doji) is the degenerate interval `[c, c]` — inside iff `band_lo ≤ c ≤ band_hi`.
- A candle body exactly spanning the band (`min(o,c) == band_lo`, `max(o,c) == band_hi`) is inside.

## Warm-up behaviour

`None` (unknown) until the stream has **two pivots of each kind** (so both labels exist — first pivots are unlabeled seeds per P1.6) — regardless of candle count. The band test additionally requires 20 closed candles (`band_warm`); before that, rules 3–5 still classify. Unknown is a first-class output consumed as "metric inactive" (the platform's A3 refuse-until-ready convention). Restart → re-warm, like every P1 utility.

## Replay implications

All inputs (pivots, labels, candles) are pure folds of the candle stream; the classification adds no clock, no randomness, no memory — identical stream → identical state sequence, and the prefix/no-repaint property is inherited unchanged (a state at candle *n* depends only on candles ≤ *n*). The state sequence becomes part of what P1.21's determinism harness indirectly covers through its consumers.

## Examples

Assume last H = 110 (HH), last L = 100 (HL), band = [100, 110], 20 candles observed:

1. 14 of the last 20 bodies inside [100, 110] → **RANGE** (band overrides the bullish labels).
2. 8 of 20 inside, labels HH∧HL → **BULLISH**.
3. 8 of 20 inside, labels LH∧LL → **BEARISH**.
4. 8 of 20 inside, labels HH∧LL → **RANGE** (mixed).
5. Same but the L chain still on its seed (label None) → **None** (unknown).
6. Doji at 110 exactly, band [100, 110] → its body counts inside.
