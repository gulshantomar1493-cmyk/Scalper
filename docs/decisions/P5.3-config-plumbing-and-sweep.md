# DECISION D25 — D9 config-plumbing (owed) + P5.3 config-sweep tool

**Date:** 2026-07-19 · **Status:** Accepted (final buildable roadmap item) ·
**Scope:** implementation decision only — lands the config plumbing D9 (P1.3)
recorded as owed, then builds the P5.3 calibration sweep that was blocked on
it. No architecture change; no frozen engine touched.

## Background

D9 (P1.3, `docs/decisions/P1.3-regime-constants.md`) decided the regime /
momentum-shift constants live in **config**, but pinned that the engines
ship config-blind (the ratio a constructor arg) and "config plumbing lands
with composition." Through P1–P4 that plumbing stayed owed: the engines were
composed with defaults numerically equal to the D9 pins, and every freeze
audit recorded the obligation. P5.3 (config-sweep calibration) was blocked on
it. This decision pays the debt and builds the sweep.

## D25.1 — The config plumbing

`backend/marketscalper/config.py` gains two frozen sections whose defaults
EQUAL the frozen §4.2 literals:

- `regime`: `compression_ratio=0.6`, `expansion_ratio=1.5`,
  `median_window_bars=240`
- `momentum`: `shift_accel_atr_ratio=0.1`

They follow the fixed 3-layer order (example → config.yaml → env) with an
explicit env map (`MARKETSCALPER_REGIME_*` / `MARKETSCALPER_MOMENTUM_*`) and
**refuse-to-start validation** (ratios must be finite and > 0;
`median_window_bars` a positive int) — the same fail-loud doctrine as the
DSN / equity guards.

`main.py` bridges the validated config layer to the engine's `RegimeConfig`
once in `_run` and applies it to BOTH the live pipelines and the in-app
replay wiring (via `functools.partial`), so an in-app replay reproduces live
under a calibrated config. `_StructurePipeline` / `_wire_structure_engines`
gained `regime_cfg=None` + `shift_accel_atr_ratio=0.1` params.

**Byte-identical contract:** the params default to the D9 pins, so every
caller that passes neither — the F2 replay button, the determinism harness,
every test — is byte-for-byte unchanged. Verified: the determinism V1–V4
hashes did not move, and a test pins `config.RegimeConfig()` == the engine's
`RegimeConfig()` defaults. **No `ENGINE_VERSION` bump** — this is composition
plumbing, not an engine-logic change; at default config the engines compute
exactly as before. (Config provenance vs. the D1 code stamp: a future concern
if the owner calibrates — recorded, not built.)

## D25.2 — The P5.3 config-sweep tool

`backend/marketscalper/calibration.py` — offline / read-only:

- `config_stats` (pure): joins a config run's admitted recommendations to
  their terminal lifecycle outcomes and computes the fees-included **NET**
  expectancy (`mean(eval_r − fee_in_R)`, §0 rule 4), gross expectancy, and
  win rate over `evaluated` recs only (invalidated/expired excluded).
- `rank_sweep` (pure): ranks configs by net expectancy above a minimum
  evaluated-sample floor; ties keep input order; lists every config's stats
  and names a `best_label` — or reports "inconclusive" below the floor.
- `_CaptureRecorder`: a recorder-shaped in-memory capture (record /
  record_lifecycle) so the sweep drives the EXACT production chain with no DB.
- `sweep` / `run_config` (async): replay a candle range through the real
  composition once per candidate config. **Provider-blind (P0.19):** the
  concrete `ReplayFeed` and `_wire_structure_engines` are INJECTED (the
  create_app pattern), so the module imports no provider — the import-boundary
  gate passes.

`scripts/calibration_sweep.py` is the owner-operated runner (lives in
`scripts/`, exempt from the boundary gate, so it may inject `ReplayFeed`):
loads config, fetches the D19.2 RVOL seed, sweeps a default grid around the
D9 pins, and prints the ranked report.

## D25.3 — The line the tool does NOT cross (D9 doctrine)

The tool **reports**; it never tunes. Picking a calibration from the sweep,
and marking any strategy TRUSTED, remains the owner's judgment over a real
≥200-recommendation campaign with positive net expectancy (§0 rule 4 /
P5.6-P5.8). There is no automated "best config" write-back, no adaptive loop —
consistent with the frozen "no calibration/adaptive logic" discipline.

## D25.4 — Consequences

- The regime/momentum constants are now config-driven and sweepable; P5.3 is
  unblocked and delivered.
- Remaining roadmap is now entirely owner-operational (P5.6/P5.8 live
  campaign; G4 events calendar; the manual rule-quality audits) or optional
  (Phase 6 Delta feed). No buildable code item remains.
