# DECISION P2.14 — Order Block Engine detail set (§4.5 pins)

**Date:** 2026-07-18 · **Status:** Accepted (engine-wise directive; pins per
the D9/D11/D12 precedent) · **Scope:** implementation decision only — zero
architecture change. Sole specification for the Order Block Engine
(roadmap P2.14, P2.15, P2.17 traceability).

---

## D13.1 — Trigger and source candle

- An OB is created only on a **displacement BOS** of the 1m stream:
  `BosEvent.displacement is True` (strict — `False`/`None` never create;
  the 1.2×ATR qualification already lives in the frozen BosDetector and is
  not re-computed here).
- **Source candle** = the last **opposite-color** candle strictly BEFORE
  the BOS bar, scanning back at most **20 bars** (bound explicitly
  arbitrary, P5-owned). BOS UP → last bearish candle (`c < o`) → **bullish
  OB**, zone `[low, open]`; BOS DOWN → last bullish candle (`c > o`) →
  **bearish OB**, zone `[open, high]` (§4.5 verbatim). Dojis (`c == o`)
  have no color and are skipped. No opposite candle within the bound → no
  OB for that BOS.
- **Identity** = (direction, source-candle ts); duplicate detections are
  ignored.

## D13.2 — Lifecycle (§4.5 verbatim, boundaries pinned)

Per closed 1m candle, existing zones first (a zone is never evaluated on
its own creation bar), **break checked before mitigation**:
- **broken**: close strictly through the far side — bullish zone
  `close < zone_lo`; bearish zone `close > zone_hi`. Boundary equality is
  not a break.
- **mitigated** (from active only): first candle whose range overlaps the
  zone — `low ≤ zone_hi AND high ≥ zone_lo` (inclusive). Mitigated is
  sticky; mitigated → broken remains possible. Only UNMITIGATED blocks are
  tradeable (weight 0 when mitigated — scoring is P3's concern; the engine
  exposes status only).

## D13.3 — Breaker transition (P2.17)

A **non-breaker** OB that breaks spawns a **breaker**: same zone, flipped
direction, `breaker=True`, created at the break bar, status active, and its
lifecycle starts the next bar (same rules: retest overlap → mitigated;
close strictly through the far side → broken). **One flip only** — a broken
breaker spawns nothing (no breaker-of-breaker; loop-free by construction).

## D13.4 — Bounds and persistence

- The engine tracks the most recent **10** zones per (direction, breaker)
  bucket (bound explicitly arbitrary, P5-owned); older zones drop from
  tracking (state-only history, stale by definition).
- Persistence capability only (R1): OBs map to the schema kinds
  `OB_BULL`/`OB_BEAR` with `p1 = zone top`, `p2 = zone bottom` (§3 comment
  order) and lifecycle statuses `active`/`mitigated`/`broken` — an exact
  vocabulary fit via the existing helpers. **Breakers are state-only**
  (the frozen `levels.kind` vocabulary has no breaker value — channels/PWH
  precedent); the row helper rejects them loudly.

## D13.5 — Cadence (composition pin)

1m closed candle: frozen structure chain (…`bos.update`) →
`ob.on_bos(event if displacement)` → … existing cadence … →
`ob.update(candle)` (after the liquidity engine) → payload. The OB engine
holds its own bounded 21-candle history (pure consumer: candles + BOS
events; nothing from other engines is re-computed).

## Freeze-audit addenda (recorded rulings + fixes)

- **Broken zones drop immediately** after spawning their breaker: the audit
  proved re-detection of a pruned identity is impossible (same-direction
  source timestamps strictly increase; a pruned source is provably outside
  every future scan window), so retaining broken zones only evicted live
  zones from the D13.4 buckets. "10 zones per bucket" therefore means 10
  UNBROKEN zones.
- **Stale-latch armor:** `update()` consumes a pending BOS only when
  `bos.ts == candle.ts` — a latch surviving a mid-cadence exception (the
  feed's reconnect path swallows engine exceptions) is discarded, keeping
  live behavior aligned with replay.
- Dedup lifetime is bounded to tracked blocks (sound per the proof above);
  the audit also formally verified: degenerate zero-width zones are
  impossible, duplicate breaker identities are impossible, the duplicate
  path leaks no state, and no set/dict iteration order reaches any output.

## Consistency check

§4.5's formulas are transcribed verbatim; only the scan bound, tracking

fallback for missing schema vocabulary, capability-only persistence).
